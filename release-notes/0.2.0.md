### Brush Engine V2

- Added a modular, deterministic S Pen engine for **Ink** and **Eraser** with the restored original v0.1.1 engine retained as an immediate fallback.
- Added canonical stylus samples, time-aware position and pressure filtering, bounded midpoint-quadratic paths, arc-length dab placement, explicit ink/eraser commands, deterministic recording, and trace replay.
- Added **Direct**, **Balanced**, **Smooth**, and bounded **Custom** tuning profiles.
- Added continuous variable-width **Ribbon** coverage while retaining discrete **Dabs** coverage for comparison and compatibility.
- Added guarded pressure-to-radius continuity so isolated pressure excursions cannot create implausible width bulges.
- Added strict pen-down and pen-up handling that preserves taps and short strokes while excluding displaced contact coordinates.

### Spike and bridge protection

- Added stale-session termination so a missed Android release event cannot connect two separate physical strokes.
- Added mid-stroke discontinuity segmentation. A confirmed coordinate relocation starts a new subpath instead of drawing across the canvas.
- Added chronological coalesced-event normalization, cross-batch deduplication, foreign-pointer rejection, stale-sample rejection, and a coordinate transform captured once at pen-down.
- Added an explicit `strokeId` boundary at the renderer.
- Added a final pixel-stage ribbon gap limit. Even if upstream input or state is corrupted, the rasterizer paints a new cap instead of a long connecting line.

### Production and diagnostics

- Debug APKs retain tuning, import/replay/export, synchronized native MotionEvent telemetry, sanitized WebView events, and final V2 samples.
- Release APK/AAB builds disable native telemetry and raw event retention while keeping bounded brush controls and the Original fallback.
- Fresh release-candidate installs use V2 for supported S Pen brushes; a saved explicit Original/V2 choice is preserved across updates.
- Mouse, touch, and unsupported specialty brushes continue through the original engine.

### Signed Android release pipeline

- Replaced the debug-signed GitHub release workflow with a fail-closed signed release pipeline.
- Produces a verified release APK and Android App Bundle from the same versioned source.
- Verifies APK signing with `apksigner`, verifies AAB signing with `jarsigner`, and publishes SHA-256 checksums.
- Separates debug and release web-asset generation so production cannot accidentally package debug telemetry configuration.
- Release packaging now fails when the real keystore, alias, or passwords are missing instead of silently using Android's debug certificate.
