### Tablet-first controls and visual hierarchy

- Modernized the production WebView controls with larger 48 px targets and 56 px coarse-pointer targets for Galaxy Tab touch and S Pen use.
- Added persistent selected states, clearer disabled states, keyboard focus rings, press feedback, reduced-motion handling, and stronger contrast across radial tools, modal actions, and project controls.
- Refined the Tablet Command Deck with clearer mode grouping, status surfaces, transport controls, accessible labels, and visible Draw, Frames, Layers, and Actions states.
- Added stable two-finger canvas pan and 0.35×–8× pinch zoom anchored under the gesture centroid, without accidental rotation from changing finger angles.
- Kept S Pen drawing immediate while briefly arbitrating finger input so a pinch cannot leave an unwanted first-contact mark.

### Project-wide Static Background

- Added one shared editable Static Background behind every animation frame without duplicating it into frame artwork.
- Integrated Static Background through live drawing, playback, onion skin, thumbnails, eyedropper, A/B comparison, PNG/GIF/video export, autosave, project archives, and schema migrations.
- Added Static Background controls to the Layer Workspace with structural lockout and undo/redo-safe editing.

### Contextual tablet workspaces

- Added the contextual Timeline Workspace for frame navigation, selection, timing operations, playback, and S Pen-oriented animation control.
- Added the contextual Layer Workspace for layer visibility, ordering, naming, duplication, deletion, blend control, and Static Background management.
- Added the Tablet Command Deck for quick access to Draw, Frames, Layers, Actions, playback, Brush Lab, and workspace collapse.

### Offline testing and release safety

- Added a fully offline feedback report that captures reproducible device and project context without transmitting artwork or personal data.
- Upgraded GitHub Actions to the Node 24 runtime and added fail-closed release-note, release-policy, documentation, signing, APK, and AAB validation.
- Release AABs require the permanent Android upload key; disposable CI keys remain verification-only and cannot be mistaken for Google Play artifacts.

### Free and private by design

- InkFrame remains free, offline-first, ad-free, subscription-free, account-free, and MIT licensed.
- Production builds exclude native diagnostics and raw trace tooling.
