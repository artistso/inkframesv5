### Native S Pen production canvas

- Integrated the physically validated Kotlin/HWUI S Pen renderer into the complete Glass Horizon studio rather than replacing the application with a reduced canvas.
- Preserved the framed canvas, perimeter timeline, circular timeline, radial controls, projects, layers, animation, exports, square/circular canvases, and existing project format.
- Added typed Kotlin context validation so every native stroke remains bound to its original project, frame, layer or Static Background, canvas geometry, and brush state.
- Added a native hover cursor with brush-diameter, color, and eraser preview plus a Kotlin-derived frame/layer/hold status pill.

### Canvas navigation and artist workspace

- Hid the engineering Brush Engine strip from the normal artist workspace while retaining Brush Lab through the Control Deck.
- Added Fit, centered pan/zoom, exact 1:1 actual-pixel inspection, Hand mode, viewport gesture guidance, and a geometry-only canvas navigator for zoomed detail work.
- Kept navigator and hover systems outside artwork, history, persistence, and the S Pen hot path.

### Kotlin production architecture

- Added read-only Kotlin mirrors for studio context, project/frame/layer reconciliation, active cel addressing, playback range, selections, holds, and timeline exposure.
- Added bounded schema-2 stroke envelopes and stale-context rejection before JavaScript history or artwork mutation.
- Began the Kotlin 2 / API 36 production toolchain migration while retaining JVM and Android regression coverage.

### Android and Google Play release

- Advances the Android package to version 0.5.0 with a committed Play version code and target API 36.
- Produces signed APK and Android App Bundle artifacts through the fail-closed upload-key pipeline.
- Keeps native diagnostics, raw traces, the engineering laboratory launcher, and experimental front-buffer binaries out of production artifacts.
- InkFrame remains free, offline-first, ad-free, subscription-free, account-free, and MIT licensed.
